## 1. 프로젝트 개요

### 해결하려는 문제

매일 아침 시장이 열리기 전, 투자자는 전일 미국 시장 흐름·지표·뉴스를 종합해 오늘 코스피 방향을 스스로 판단해야 한다. 이 작업은 반복적이고 시간이 걸리며, 감정적 판단이 개입되기 쉽다.

### 솔루션 한 줄 정의

> **"매일 아침 8시 30분, AI가 시장 데이터를 수집·분석해 코스피 방향 예측과 추천 종목을 텔레그램으로 전송하고, 웹에 아카이빙한다."**

```

    ▼         ▼

    │
```

| **코스피 브리핑** | 월~금 08:30       | 한국 시장 개장일 | 전일 미국 마감 기반 코스피 시초가 방향 예측     |

## 4. 기술 스택 및 선택 이유

### 4-1. 실행 환경: GitHub Actions

| **항목**    | **내용**                                          |
| --------- | ----------------------------------------------- |
| **선택 이유** | 별도 서버 없이 무료로 크론 스케줄 실행 가능                       |
| **대안 검토** | macOS launchd (기기 켜져 있어야 함), 원격 트리거 (인증 문제로 제외) |
| **한계**    | 무료 플랜 2,000분/월 (브리핑 1회 약 2분 → 월 60회, 충분)        |

### 4-2. AI 엔진: Claude CLI

```bash
claude --dangerously-skip-permissions -p "$(cat agents/kospi_morning.md)"
```

- 에이전트 프롬프트 파일(`.md`)을 읽어 Claude가 직접 파일 읽기·쓰기·웹 검색·코드 실행을 수행
- `--dangerously-skip-permissions`: CI 환경에서 인터랙션 없이 자동 실행
- **인증:** `ANTHROPIC_API_KEY` 환경변수로 주입 (GitHub Secrets)

### 4-3. 웹 서비스: Netlify + 정적 HTML

- GitHub `main` 브랜치에 push되면 Netlify가 자동으로 `web/` 폴더를 배포
- 서버리스 — 백엔드 없음, 순수 정적 HTML/CSS/JS
- 브리핑 생성 후 GitHub Actions이 HTML을 커밋·푸시 → Netlify 자동 배포

### 4-4. 알림: 텔레그램 봇

- Python 스크립트가 Telegram Bot API를 직접 호출 (외부 라이브러리 불필요)
- 메시지: 핵심 요약 300자 이내 + 웹 링크

## 5. 에이전트 설계 철학

### "Claude가 기자이자 애널리스트"

에이전트 프롬프트(`.md` 파일)는 Claude에게 아래 역할을 부여한다:

```
1. 데이터 수집가  → yfinance + 웹 검색으로 실시간 지표 수집
2. 애널리스트     → 수집 데이터 기반 방향 예측 + 근거 작성
3. 종목 전략가    → 잭 켈로그 20일선 전략으로 종목 선별
4. 퍼블리셔       → HTML 파일 + 텔레그램 메시지 파일 저장
```

### 핵심 설계 원칙

| **원칙**      | **구현 방식**                                   |
| ----------- | ------------------------------------------- |
| **실시간 데이터** | 학습 데이터 사용 금지, 반드시 웹 검색으로 현재가 확인             |
| **장애 격리**   | 각 Step 실패해도 다음 Step 계속 진행                   |
| **중복 방지**   | 텔레그램 전송은 workflow 단에서 1회만 (에이전트에서 직접 전송 금지) |
| **휴장일 판단**  | workflow에서 처리, 에이전트 내 중복 확인 제거              |

### 레이아웃 구조

```
┌─────────── GNB (네비게이션 바) ───────────┐
│  Daily30'    날짜·시각   [코스피]  [🌙]   │
└───────────────────────────────────────────┘
┌──────────────────────┬────────────────────┐
│                      │   시장 지표  - LIVE │
│  예측 카드           │                    │
│  ├ 방향 예측         │  [미국 시장]    ▼  │
│  ├ 상승/하락 확률    │  나스닥  17,697    │
│  └ 신뢰도            │  SOX     4,833 ⬆   │
│                      │  ···               │
│  예측 근거           │  [한국 시장]    ▼  │
│  ├ bullet ×5+        │  코스피  2,583     │
│  └ 리스크 요인       │  ···               │
│                      │  [변동성]       ▼  │
│  잭 켈로그 종목      │  WTI    $61.60     │
│  ├ 종목 1~5          │  공포탐욕: 35 공포 │
│  └ 실행 가이드       │  [반원 게이지]     │
│                      │                    │
│  과거 브리핑 아카이브│                    │
└──────────────────────┴────────────────────┘
```

### 반응형 대응

- **데스크톱(900px+):** 2컬럼 — 브리핑 본문 + 우측 시장 지표 패널 sticky
- **모바일(900px-):** 1컬럼 — 브리핑 본문 → 시장 지표 (하단)

### 시장 지표 사이드바

- **Sparkline 차트:** Canvas API로 최근 10개 데이터 포인트 면적 그래프
- **공포탐욕 게이지:** 반원 Canvas 게이지 + 전일/1주/1달/1년 이력
- **데이터 소스:** 에이전트가 `window.MARKET_DATA = {...}` 형태로 HTML에 직접 주입

### 테마

- **기본값:** 라이트 모드
- **전환:** GNB 우측 토글 버튼 (🌙 ↔ ☀️)
- **지속성:** `localStorage`에 저장

## 7. 데이터 흐름 상세

```
GitHub Actions
    │
    ├─ python3 scripts/fetch_data.py --type kospi
    │      └─ yfinance로 전일 종가 데이터 수집
    │         → data/latest_kospi.json 저장
    │
    ├─ claude --dangerously-skip-permissions -p "$(cat agents/kospi_morning.md)"
    │      └─ Claude가 수행하는 작업:
    │         1. data/latest_kospi.json 읽기
    │         2. 웹 검색 (부족한 데이터 보완, 종목 현재가 확인)
    │         3. 분석 → data/analysis_kospi.json 저장
    │         4. HTML 생성 → web/briefings/YYYY-MM-DD-kospi.html 저장
    │                      → web/index.html 덮어쓰기
    │         5. 텔레그램 메시지 → data/telegram_message_kospi.txt 저장
    │
    ├─ python3 scripts/send_telegram.py --type kospi
    │      └─ telegram_message_kospi.txt 읽기
    │         → Telegram Bot API 호출
    │         → 실패 시 analysis_kospi.json 폴백 메시지 생성
    │
    └─ git add web/ data/ → git commit → git push
           └─ Netlify 자동 감지 → web/ 폴더 재배포
```
